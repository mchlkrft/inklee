## 4. Favicon still Vercel logo in Google  [MAIN]
Files: `apps/web/src/app/layout.tsx` (metadata, NO `icons` key currently), `apps/web/src/app/{favicon.ico,icon.svg,apple-icon.png}` (already exist, spiderweb brand).
Root cause(s) per the Reddit thread + Google favicon docs:
- Google caches favicons aggressively and recrawls slowly (weeks). It looks for `/favicon.ico` at site root and a `<link rel="icon">` in the homepage `<head>`. Google support for SVG favicons is weak; it prefers a real .ico/.png that is a multiple of 48x48.
- The "Vercel logo" is Google's stale cached fallback from before brand icons were added/declared.
Fixes:
- Add explicit `icons` to `metadata` in layout.tsx:
  ```ts
  icons: {
    icon: [
      { url: "/favicon.ico", sizes: "any" },
      { url: "/icon.svg", type: "image/svg+xml" },
    ],
    apple: "/apple-icon.png",
  },
  ```
- Ensure `favicon.ico` contains a 48x48 (or multiple) entry. Current ico = 16+32 only. Regenerate from `public/branding/illustrations/spiderweb/spiderweb-mustard.svg` to include 48x48/96x96 (use sharp + to-ico, or the existing spiderweb gen script `.scratch/make-spiderweb-gif.cjs` pattern). Verify with a tool.
- Optionally add a `manifest.ts` (web app manifest) with name + icons (helps Google + PWA).
- After deploy: in Google Search Console request re-indexing of the homepage; favicon refresh can lag days/weeks. Document this as the non-code follow-up.
- [x] metadata.icons added in layout.tsx (favicon.ico + /icon.png 96 + icon.svg + apple)
- [x] favicon.ico regenerated with 16/32/48 (PNG-embedded ICO) from src/app/icon.svg; new public/icon.png 96x96. Script: scratchpad/gen-favicon.cjs (hand-rolled ICO encoder, sharp via absolute path).
